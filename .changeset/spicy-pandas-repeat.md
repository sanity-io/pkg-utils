feat: initial release of `@sanity/vanilla-extract-tsdown-plugin`, a tsdown plugin for vanilla-extract that extracts CSS into a single `lightningcss`-optimized file, following the same architecture and option vocabulary (`fileName`, `minify`, `target`, `inject`) as `@tsdown/css`. Like `css.inject`, `inject` is disabled by default and `inject: true` injects a relative CSS import; `inject: {nodeCompat: true}` wires up the whole conditional CSS export pattern instead - the self-referential import, the no-op JS shim, and the conditional `"./<fileName>"` export written to `package.json` through the plugin's `tsdownConfig` hook when tsdown's `exports` feature is enabled. Unlike `@vanilla-extract/rollup-plugin` it doesn't declare `rollup` as a peer dependency, and it declares plugin hook filters ([vanilla-extract#1641](https://github.com/vanilla-extract-css/vanilla-extract/issues/1641)) so rolldown skips the Rust ↔ JS roundtrip for modules that aren't vanilla-extract related
